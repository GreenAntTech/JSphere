Why JSphere

GitHub as your runtime: deploy with Git references, not pipelines.

Problems we solve

  • DevOps friction for simple deploys and previews
  • Multi‑repo coordination headaches
  • Slow rollbacks and environment drift
  • Redundant artifact/package infrastructure

GitHub‑native approach

  • Projects = config repo + one or more package repos
  • app.json drives packages, routes, extensions, directives, settings, featureFlags
  • JSphere serves client files and server endpoints directly
  • Deploy by updating packages.<name>.reference to a branch/tag/commit

Real‑world scenarios

  • Rapid prototyping: idea → working preview in hours
  • Enterprise microservices: independent deploys, instant rollbacks
  • Open‑source: feature branches are live testable, tags for releases
  • Education/content: global updates by switching references

Benefits

  • Zero heavy pipeline overhead
  • Version control = deployment control
  • Instant collaboration and previews
  • Lower costs and simpler operations

Getting Started

Prereqs: Deno installed; GitHub account + Personal Access Token (repo scope).

1) Install the JSphere CLI

deno install -A -r https://raw.githubusercontent.com/GreenAntTech/JSphere/main/cli.ts
js version

2) Create your first project

js create-project

Follow the prompts:

  • Project name → e.g. myproject
  • Project namespace → your GitHub account/org
  • Project auth token → your GitHub PAT

This creates two repos: .myproject (config) and myproject (app), and adds an entry to your local jsphere.json.

3) Start the server and load the project

# start server (choose config with --list if needed)
js start
# or
js start --list

# load your project configuration
js load myproject

Open http://localhost to view the sample app.

4) Checkout repos for local editing

# checkout all packages of the loaded project
js checkout

# or checkout a single package
js checkout myproject

Make changes locally, then reload:

js reload

Next steps

  • Install element.js for client UI: js install-element
  • Install TS definitions: js install-types
  • Learn routing and APIs in Project Structure
  • Try the shipping workflow: Ship Features
